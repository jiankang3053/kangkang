# KangkangWeather v3.0.0

## 重点更新

- 新增自动发送环境守门器：真实发送前会检查 Windows 是否处于可交互桌面、微信是否可访问、权限是否一致。
- 支持“息屏但不睡眠”的省电运行模式检查：屏幕可以关闭，但电脑不能睡眠、锁屏或注销。
- 固定时间日报新增 180 分钟补偿窗口：到点时环境不可用会等待恢复，不再只靠 70 秒触发窗口。
- 固定日报不再被普通轮询运行时间段挡住：例如设置 06:00 就会按固定发送点处理。
- 新增定时任务医生接口：检查/修复当前用户交互式登录任务，避免跨电脑路径失效。
- Web 控制台新增“自动发送环境 / 息屏睡眠检查 / 定时任务医生”入口。
- CLI 新增 `readiness`、`power-status`、`power-apply`、`scheduler-status`、`scheduler-repair`。

## 已知边界

- 不能在电脑睡眠、注销、无交互桌面时直接控制微信发送。
- 锁屏时不强行操作微信；固定日报会进入补偿队列，恢复可用后再发。
- 微信和 KangkangWeather 建议都用普通用户权限运行，避免 UIAutomation 被权限隔离。
- 仍只支持官方 Windows 微信客户端，不使用 Hook、协议登录或后台发消息接口。
